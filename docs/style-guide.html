Design Rules

Layout: Mobile-first, max-width 420px. Hero + compact pattern: primary route gets visual dominance, fallbacks are compact cards below.

Spacing: All dashboard sections (hero card, journey, departures, alternatives) use gap: var(--space-xl) (24px) for uniform vertical rhythm. Top bar to first content also uses 24px. Internal card padding uses var(--space-xl). Section titles use margin-bottom: var(--space-md) (12px) to their content. Never use ad-hoc margins between sections — always rely on the parent flex gap.

Theme: Dark mode default, system-adaptive toggle. All colors defined as CSS custom properties for instant theme switching.

Countdown: Large countdown is the primary element, centered in hero card. Absolute times are secondary, shown in equal-width grid cells below.

Hero card flow: Reads top to bottom: 1) Profile + route pill, 2) Countdown + status (centered), 3) Time grid (Leave by | Scheduled | ETA), 4) Stop code + name (centered), 5) Realtime badges (centered). All text centered except profile name row.

Status: Green (>10 min) → Yellow (3-10 min) → Red (<3 min) → Purple (grace period) → Grey (missed). Applied via background tint + text color, never just color alone.

Transit lines: Stop codes are color-coded with the route's official color (text color, not background). Provides instant recognition without reading text.

Typography: DM Sans for UI, DM Mono for times, stop codes, and countdowns. Bold for emphasis, never underline or italic.

Animations: Minimal and purposeful. Page transitions: fade-out 150ms + fade-in 200ms with subtle vertical shift (View Transitions API). Menu sheet: slide-up/down 250ms with cubic easing + backdrop fade 200ms with blur. Countdown tick on timer. No decorative motion.

Navigation: Bottom tab bar, 3 tabs: Dashboard, Profiles, Settings. Thumb-friendly zone.

Interaction: Tap departure in list to update hero card. Pull-to-refresh on dashboard.
